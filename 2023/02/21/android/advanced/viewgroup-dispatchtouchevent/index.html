Execution flow

  1. Checks whether a sequence of input events is self-consistent.

  2. Filter the touch event to apply security policies.

  3. Handle an initial down.

  4. Check for interception.

  5. Check for cancelation.

  6. Distribute the ACTION_DOWN event
