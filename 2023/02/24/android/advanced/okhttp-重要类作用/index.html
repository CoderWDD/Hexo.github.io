OkHttpClients

Factory for calls, which can be used to send HTTP requests and read their responses.
OkHttpClients Should Be Shared

  • 使用 建造者模式 来构造 OkHttpClient 对象，可以选择性地设置个性化设置

  • 由 OkHttpClient.Builder 源码定义，可以看出，OkHttpClient 是一个核心的管理类，其拥有的网络请求中的 Dispatcher、ConnectionPool 、Interceptor 等

Request

An HTTP request. Instances of this class are immutable if their body is null or itself immutable.

Response

An HTTP response. Instances of this class are not immutable: the response body is a one-shot value that may be consumed only once and then closed. All other properties are immutable.
This class implements Closeable. Closing it simply closes its response body. See ResponseBody for an explanation and examples.

Dispatcher

Policy on when async requests are executed.
Each dispatcher uses an ExecutorService to run calls internally. If you supply your own executor, it should be able to run the configured maximum number of calls concurrently.

RealCall

Bridge between OkHttp's application and network layers. This class exposes high-level application layer primitives: connections, requests, responses, and streams.
This class supports asynchronous canceling. This is intended to have the smallest blast radius possible. If an HTTP/2 stream is active, canceling will cancel that stream but not the other streams sharing its connection. But if the TLS handshake is still in progress then canceling may break the entire connection.

是 Call 的实现类，对建立连接、发送请求、读取响应、IO流进行了封装

负责构造 内部责任链，并执行

真正发出请求并把整个请求过程组织起来的类

  • 对于 同步执行 的任务，调用 execute()，对于 异步执行 的任务，调用 enqueue()

  • 不管是 同步 还是 异步，最后都会进到 getResponseWithInterceptorChain() 进行最终调度
