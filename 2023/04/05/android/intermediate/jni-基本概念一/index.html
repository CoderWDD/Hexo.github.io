什么是JNI

JNI 全称为 Java Native Interface，我们在 Android 开发中，现在一般都是基于 Java/Kotlin 进行开发，但是，我们难免需要与用 C/C++ 编写的本地方法进行交互，而我们知道，Java/Kotlin 本身并不能直接调用 C/C++ 的代码，所以，就需要用到 JNI，也就是说 JNI 就像是连接 C/C++ 与 Java/Kotlin 之间的桥梁，使得 Java/Kotlin 能间接地调用到 C/C++ 的代码

怎么使用 JNI

  • 安装 CMake 和 NDK
  • 创建 native C++ 项目
  • 运行

安装 CMake 和 NDK

打开 Android Studio 的 setting 面板，找到 Appearance&Behavior -> System Settings -> Android SDK -> SDK Tools，勾选上 NDK 和 CMake，进行下载安装即可

创建 native C++ 项目

打开 new Project 面板，拉到最下面，选择 native C++，选择响应的 C++ 版本并创建就行

JavaVM and JNIEnv

先来看看 Google 官方的解释

JNI defines two key data structures, “JavaVM” and “JNIEnv”. Both of these are essentially pointers to pointers to function tables. (In the C++ version, they’re classes with a pointer to a function table and a member function for each JNI function that indirects through the table.)

The JavaVM provides the “invocation interface” functions, which allow you to create and destroy a JavaVM. In theory you can have multiple JavaVMs per process, but Android only allows one.

The JNIEnv provides most of the JNI functions. The JNIEnv is used for thread-local storage.

JavaVM

从官方描述中可以看出，理论上，一个进程，是可以有多个 JavaVM 的，但是 在 Android 中，只允许一个进程拥有一个 JavaVM

JavaVM 是一个代表 Java虚拟机实例 的数据结构，具体是一个 struct 类型，可以用于获取 JVM实例 的引用，并通过它，访问一下 JVM 的函数，比如创建 Java 对象、调用 Java 方法、获取当前线程的 JNIEnv 指针等，下面来看看其具体定义：
