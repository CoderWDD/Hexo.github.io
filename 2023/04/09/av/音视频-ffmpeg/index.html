什么是 FFmpeg

FFmpeg 是一套开源的跨平台音视频处理工具集，可以实现音视频的 编解码、转码、处理、播放 等功能。

FFmpeg 支持几乎所有的音视频编解码格式，包括常见的 MP4、AVI、MKV、FLV、WMV、MOV、MP3、AAC 等格式，也支持一些专业的视频格式和音频格式。

FFmpeg 是跨平台的，因此可以在各种操作系统上使用，例如 Windows、macOS、Linux 等。同时，FFmpeg 还提供了各种语言的 API 和库，包括 C语言、C++、Python、Java、Swift 等，方便开发者集成和调用 FFmpeg 的功能。

FFmpeg 的组成库

libavcodec： provides implementation of a wider range of codecs.

  • 功能：
    • 可以实现音视频编解码的功能，提供了丰富的音视频编解码库和 API，可以支持多种音视频格式的编解码，例如 H.264、AAC、MP3、VP9、AV1 等。
  • 特点：
    • 支持多种音视频格式的编解码：比如支持 H.264、AAC、MP3、VP9、AV1 等格式

libavformat： implements streaming protocols, container formats and basic I/O access.

  • 功能：
    • 可以实现音视频文件的读取、写入、封装、解封装等操作，同时还可以对音视频文件进行转换、拼接、剪辑等操作
  • 特点：
    • 支持多种音视频格式的 解封装 和 封装

libavutil： includes hashers, decompressors and miscellaneous utility functions.

基础工具库，提供了各种常用的工具函数和数据结构，供 FFmpeg 其他组件调用

  • 功能包括：
    • 数据结构：包括链表、队列、堆、哈希表、树等数据结构，为音视频处理提供了基础的数据结构支持。
    • 算法：包括排序、查找、编码、解码、加密、解密等算法，为音视频处理提供了基础的算法支持。
    • 数学：包括矩阵、向量、统计、概率等数学工具，为音视频处理提供了基础的数学支持。
    • 字节操作：包括位操作、字节序转换、字节流拼接等工具，为音视频处理提供了基础的字节操作支持。
    • 日志输出：包括错误日志、调试日志、统计日志等功能，为音视频处理提供了基础的日志输出支持。
  • 特点：
    • 高效、可靠、可移植
    • 高度模块化
    • 易于扩展和集成

libavfilter： provides means to alter decoded audio and video through a directed graph of connected filters.

  • 功能：
    • 实现音视频滤镜处理的功能，例如字幕、变速、变调、颜色调整、降噪、模糊、马赛克等
  • 特点：
    • 支持多种音视频滤镜的处理：变速、变调、颜色调整、降噪、模糊、马赛克等
    • 可以有多个输入和多个输出

libavdevice： provides an abstraction to access capture and playback devices.

  • 功能：
    • 实现支持多种音视频设备的 采集和输出 的功能，例如摄像头、麦克风、扬声器、音频接口等
  • 特点：
    • 支持多种音视频设备的采集和输出

libswresample： implements audio mixing and resampling routines.

  • 功能：
    • 实现音频重采样的功能
    • 可以支持多种音频格式之间的转换，例如从 48kHz 转换为 44.1kHz、从双声道转换为单声道
  • 特点：
    • 支持多种音频格式之间的转换

libswscale： implements color conversion and scaling routines.

  • 功能：
    • 实现视频像素格式和尺寸的缩放和转换的功能，支持多种视频像素格式之间的转换和多种视频尺寸之间的缩放，例如从 1080p 转换成 720p 或者 480p 等的缩放
  • 特点：
    • 支持多种视频像素格式之间的转换：包括 YUV420P、YUV422P、RGB24、RGBA 等
    • 支持多种视频尺寸之间的缩放：包括缩小、放大、裁剪等

FFmpeg 语法

最详细权威的用法，请查看 官方文档

基本语法格式

ffmpeg [global_options] {[input_file_options] -i input_url} ... {[output_file_options] output_url} ...

  • 命令组成介绍：
    • global_options：全局选项，包括 FFmpeg 的一些通用参数和选项，例如日志级别、硬件加速、线程数等。
    • input_file_options：输入文件选项，包括 FFmpeg 对输入文件的一些处理参数和选项，例如输入格式、采样率、分辨率、视频编解码器、音频编解码器等。
    • -i input_url：输入文件的URL，用于指定 FFmpeg 要处理的输入文件。可以是本地文件路径，也可以是HTTP、FTP、RTSP等网络流媒体协议的URL地址。
    • output_file_options：输出文件选项，包括 FFmpeg 对输出文件的一些处理参数和选项，例如输出格式、输出比特率、输出分辨率、视频编解码器、音频编解码器等。
    • output_url：输出文件的 URL，用于指定 FFmpeg 处理后的输出文件。可以是本地文件路径，也可以是 HTTP、FTP、RTSP 等网络流媒体协议的 URL 地址。
  • 不同的括号作用：
    • 方括号 [] 表示选项是可选的，可以选择性地添加或省略该选项，如果省略，则使用默认值或 FFmpeg 自动检测的值。例如，[output_file_options] 表示指定输出文件选项是可选的
    • 花括号 {} 表示选项是必选的，必须从花括号内的选项中选择一个或多个选项进行指定。例如，{[input_file_options] -i input_url} 表示必须选择一个或多个输入文件选项，并指定一个输入文件的 URL

音频选项

Option 描述
-aframes number 设置输出的音频帧数
-aq quality 设置音频质量（编解码器特定）
-ar rate 设置音频采样率（以Hz为单位）
-ac channels 设置音频通道数
-an 禁用音频
-acodec codec 强制指定音频编解码器（使用’copy’表示复制流）
-ab bitrate 音频比特率（请使用-b:a选项）
-af filter_graph 设置音频滤镜
-s size 设置帧大小（WxH或缩写）
-sn 禁用字幕
-scodec codec 强制指定字幕编解码器（使用’copy’表示复制流）
-stag fourcc/tag 强制指定字幕tag/fourcc
-fix_sub_duration 修正字幕持续时间
-canvas_size size 设置画布大小（WxH或缩写）
-spre preset 将字幕选项设置为指定的预设

视频选项

Option 描述
-vframes number 设置输出的视频帧数
-r rate 设置帧率（以Hz值、分数或缩写表示）
-fpsmax rate 设置最大帧率（以Hz值、分数或缩写表示）
-s size 设置帧大小（WxH或缩写）
-aspect aspect 设置宽高比（4:3、16:9或1.3333、1.7777）
-display_rotation angle 设置流的纯逆时针旋转角度
-display_hflip 设置流的水平翻转（如果未设置，则覆盖任何显示旋转）
-display_vflip 设置流的垂直翻转（如果未设置，则覆盖任何显示旋转）
-vn 禁用视频
-vcodec codec 强制指定视频编解码器（使用’copy’表示复制流）
-timecode hh:mm:ss[:;.]ff 设置初始时间码值
-pass n 选择通过数（1到3）
-vf filter_graph 设置视频滤镜
-b bitrate 视频比特率（请使用-b:v选项）
-dn 禁用数据

文件选项

Option 描述
-f fmt 强制指定输出格式
-c codec 指定编解码器
-codec codec 指定编解码器
-pre preset 指定预设
-map_metadata outfile[,metadata]:infile[,metadata] 从输入文件中设置输出文件的元数据信息
-t duration 记录或转码”duration”秒的音频/视频
-to time_stop 转码停止时间
-fs limit_size 设置文件大小限制（以字节为单位）
-ss time_off 设置开始时间偏移
-sseof time_off 设置相对于文件结尾的开始时间偏移量
-seek_timestamp 启用/禁用通过时间戳进行跳转（使用-ss选项）
-timestamp time 设置录制时间戳（’now’表示设置当前时间）
-metadata string=string 添加元数据
-program title=string:st=number… 添加带有指定流的程序
-target type 指定目标文件类型（”vcd”、”svcd”、”dvd”、”dv”或”dv50”，可以使用可选的前缀”pal-“、”ntsc-“或”film-“）
-apad 音频补齐
-frames number 设置要输出的帧数
-filter filter_graph 设置流过滤器图
-filter_script filename 从文件中读取流过滤器图描述
-reinit_filter 在输入参数更改时重新初始化过滤器图
-discard 丢弃
-disposition 布置

全局选项

与上面的选项不同，这里的选项，是会影响整个程序最后的结果的，而不是只影响某一个文件

Option 描述
-loglevel loglevel 设置日志级别
-v loglevel 设置日志级别
-report 生成报告
-max_alloc bytes 设置单个分配块的最大大小
-y 覆盖输出文件
-n 永不覆盖输出文件
-ignore_unknown 忽略未知的流类型
-filter_threads 非复杂过滤器线程数
-filter_complex_threads -filter_complex的线程数
-stats 在编码期间打印进度报告
-max_error_rate 设置解码错误率的最大值（0.0：无错误，1.0：100％错误），超过此值FFmpeg将返回错误而不是成功。

工作流程

  • 解封装：FFmpeg 会读取输入文件中的音视频流，并将其解封装为各自的编解码器所需的数据格式。这个过程可以通过指定输入文件格式来实现。
  • 解码：FFmpeg 将音视频流中的原始数据解码为 PCM 音频或像素数据。解码后的数据可以通过指定的输出编解码器进行编码。
  • 编码：FFmpeg 将解码后的音视频数据编码为特定格式的音视频流。编码器通常会根据指定的输出质量和参数进行调整。
  • 过滤：FFmpeg 可以应用各种音视频滤镜来处理解码后的音视频数据。滤镜可以用于处理音频，视频或两者同时处理。
  • 封装：FFmpeg 将编码后的音视频流封装为特定的容器格式。容器格式是包含音频和视频流的文件格式，通常是 .avi、.mp4 等格式。封装器会根据指定的输出格式生成相应的容器文件。
  • 输出：FFmpeg 将封装后的音视频流输出到指定的输出文件或流中，以供后续处理或观看。
