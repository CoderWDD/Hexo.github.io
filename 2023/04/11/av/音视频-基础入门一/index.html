音视频_基础入门一

Created2023-04-11|Updated2023-04-11|音视频Basics
|Post View:

音视频组成

谈起音视频，我们的第一印象，应该是我们平时下载的 MP4、AVI、FLV、MKV、MPEG 文件，往往我们可以把这些文件放到播放器中，就可以直接播放了，而且是同时有视频画面和音频输出的，这也就意味着，我们下载的这个文件，实际上是把 音频 和 视频 都打包在一起的，播放器在解析我们的文件时，会分别解析音频和视频，然后进行合并输出

也就是说，我们音视频，其实是由 音频 和 视频 两部分组成，并将其打包进同一个文件中，这里的文件，就可以理解为容纳音频和视频的容器，而我们在对音频和视频进行编码时，其实也只是改变了放进容器里面的数据存放格式，并不会影响容器本身

处理流程

  • 解封装：将音视频文件进行解封装，得到音视频流数据。
  • 解码：将音视频流数据进行解码，得到PCM（Pulse Code Modulation）数据和视频帧数据。
  • 视频处理：对视频帧数据进行处理，包括解码、裁剪、缩放、转码、水印、滤镜等操作。
  • 音频处理：对PCM数据进行音频处理，包括音频解码、降噪、混音、音效处理等操作。
  • 合成：将处理后的视频帧数据和音频数据进行合成，得到最终的音视频流数据。
  • 编码：将合成后的音视频流数据进行编码，生成最终的音视频文件。

视频

引入

我们知道，视频其实就是一连串的帧连续播放的结果，也就是说，对于一个视频，其实是可以拆分成一连串的帧的。但是这就会引发另一个问题，如果我们对于每个帧都完整的保存起来，对于一个视频，就会保存很多帧，其占用空间肯定会很大（一个帧相当于一个图片，一连串的帧，肯定很大）。

同时，由于是连续播放的帧，相邻的帧之间，往往在某些地方，差别是不大的，如果这时候都完整得保留每一个完整的帧，很显然是没有必要的，因为完全可以让相邻的帧之间进行参考，从而减少保存的帧大小，这就是 P帧 和 B帧

而对于同一帧的画面来说，往往某些区域内的画面是相同或者相似的，对于这种情况，我们也可以进行处理，而不是完整得把这一帧画面的内一个点都完整记录下来

还有，对于一个没有必要的细节，我们也可以进行忽略，这时就可以和上面提到的一样，不用把一帧内的每一个点都完整记录下来了

压缩分类

正如上面说到的一样，完整的保存每一帧中的每一个点，很多情况下是会有冗余的，而且也是没有必要的，对于相邻的帧之间，也可以互相参考，这就引入了两个概念：帧内压缩、帧间压缩

同样的，上面也提到了，对于一些没有必要的细节，我们也可以进行忽略，而忽略本身这件事就是不可逆的，因此，又可以把压缩这件事，按照是否可逆的依据，划分为：无损压缩、有损压缩

编码原理

视频编码的原理是基于压缩编码技术，其核心思想是通过 去除视频数据中的冗余信息和无用信息，来减少视频数据的存储和传输成本。

视频编码器通常采用 运动估计、空间和时间预测、变换编码和熵编码 等技术，将视频信号转换为数字信号，并进行压缩编码。具体来说，视频编码过程主要包括以下几个步骤：

  • 帧内压缩：对视频帧内的像素数据进行压缩编码，包括色度、亮度信息的采样和量化、差分编码、熵编码等。
  • 帧间压缩：对视频帧之间的运动信息进行分析和编码，通过运动补偿来减少数据冗余，包括运动估计、运动补偿、熵编码等。
  • 变换编码：通过将空间域中的像素数据转换到频域中，并采用离散余弦变换（DCT）进行变换编码，以进一步压缩视频数据。
  • 熵编码：采用哈夫曼编码、算术编码等熵编码技术，将压缩后的视频数据进行进一步的压缩，以减小数据传输带宽。

处理流程

  • 解码：从视频轨道中读取视频数据流，并将其解码为原始像素数据。
  • 处理：对解码后的像素数据进行各种处理，如颜色空间转换、图像缩放、去噪等。
  • 编码：将处理后的像素数据编码为目标视频编码格式的数据流。
  • 封装：将编码后的视频数据流封装为目标容器格式，如MP4、FLV等。
  • 输出：将封装后的视频数据流输出到目标设备或网络中进行播放或传输。

编码格式

编码格式 简介 是否是无损压缩 应用场景 优点 缺点
H.264/AVC 目前最常用的视频编码标准之一，具有优异的压缩比和高质量的视频输出效果。 否 视频会议、流媒体服务、数字电视、蓝光光盘等领域 高压缩比、广泛支持、成熟稳定 编码解码复杂度高、延迟较大、不支持无限制的分辨率和帧率
H.265/HEVC H.264/AVC的后继者，提供更高效的视频压缩和更高质量的视频输出。 否 4K和8K分辨率视频、移动流媒体、云游戏等领域 更高的压缩比、更高的编码效率、支持HDR 编解码复杂度更高、处理时间更长、编码器优化较差
MPEG-2 一种有损视频编码标准，广泛应用于数字电视、DVD等领域。 否 数字电视、DVD等领域 良好的兼容性、支持广泛、可用于广播、存储和传输等多种场合 文件体积较大、压缩效率较低，不适合处理高质量视频
MPEG-4 一种广泛使用的视频编码标准，包括MPEG-4 Part 2和MPEG-4 Part 10（也称为H.264/AVC），支持多种媒体类型的编码和传输。 否 互联网流媒体、数字电视、移动设备、视频监控等领域 具有较高的压缩比、编码速度较快、成熟稳定 需要更高的带宽、处理时间相对较长、对源视频质量的要求较高
VP9 由Google开发的开放源代码视频编码格式，提供高效的视频压缩和高质量的视频输出，支持4K和8K分辨率的视频编码。 否 WebM、YouTube、Android平台等领域 较高的压缩效率、优秀的视频质量、良好的支持度 编解码复杂度高、性能相对较弱、不支持所有设备和浏览器
ProRes 由苹果公司推出的无损视频编解码器，支持多种分辨率的视频，广泛用于影视制作领域。 是 影视后期制作领域 无损压缩、支持高分辨率、色彩深度和色域，支持广泛，处理速度快 文件体积较大，需要更高的存储和传输带宽，只能在苹果系统上使用
DNxHD 由Avid Technology推出的无损视频编解码器，广泛用于视频编辑和后期制作领域。 是 影视后期制作领域 无损压缩、支持高分辨率、色彩深度和色域，支持广泛，处理速度快 文件体积较大，需要更高的存储和传输带宽，只能在Avid系统上使用
CineForm 一种高性能的视频编解码器，支持无损和有损压缩，广泛应用于影视后期制作领域。 是/否 影视后期制作领域 高性能、高压缩比、支持高分辨率、色彩深度和色域，可用于Windows和Mac 可能需要更高的计算资源，较新的版本可能需要较新的硬件支持
AV1 由Alliance for Open Media推出的开放源代码视频编码格式，提供高效的视频压缩和高质量的视频输出，支持4K和8K分辨率的视频编码。 否 视频流媒体、云游戏、移动设备等领域 较高的压缩效率、优秀的视频质量、良好的支持度 编解码复杂度高、性能相对较弱、对处理资源的要求较高
AV1（无损） AV1协议的无损压缩版本，提供高效的无损视频压缩和高质量的视频输出。 是 影视后期制作领域、科学研究领域等 无损压缩、支持高分辨率、色彩深度和色域，良好的支持度 编解码复杂度高，处理速度较慢，需要更高的计算资源和存储空间

编码

  • 分析和预处理：将原始视频进行分析和预处理，包括去噪、滤波、色彩空间转换等。
  • 块划分和运动估计：将视频划分为若干个宏块（macroblock），通过运动估计算法来找到每个宏块在前后帧之间的运动矢量，以便后续的帧内/帧间压缩。
  • 压缩编码：通过帧内/帧间压缩技术来压缩视频数据，包括空间域压缩（如DCT、IDCT、量化）和时域压缩（如帧内/帧间预测）等。
  • 熵编码：将压缩后的数据进行熵编码，以进一步压缩数据。
  • 打包封装：将编码后的视频数据打包封装成容器格式（如MP4、AVI、MKV等），以便后续的传输、存储和播放。

解码

  • 数据解包和解析：将压缩编码后的视频数据进行解包封装和解析，提取出压缩后的视频帧数据。
  • 熵解码：对压缩后的视频数据进行熵解码，将压缩前的视频数据还原出来。
  • 逆量化和反变换：对熵解码后的视频数据进行逆量化和反变换，将DCT系数还原为原始的像素值。
  • 运动补偿：将解码出的帧与前一帧或后一帧进行运动补偿，以还原出原始的视频帧。
  • 色彩空间转换：将YUV格式的视频帧转换为RGB格式，以便于显示或存储。

音频

引入

对于音频，我们知道是由不同的频率的声音组成的，从中学物理中，我们就已经知道声音的三要素分别为 振幅、频率、音色，而我们在对音频进行编码的时候，其实也是从使用场景出发，比如，我们往往会把人耳可听范围以外的频率的声音直接忽略掉，而在对于振幅较大的点附近，我们往往也会忽略掉一些振幅较小的点

编码原理

音频编码的原理基于数字信号处理技术和信息论，其核心思想是通过 去除音频数据中的冗余信息和无用信息，来减少音频数据的存储和传输成本。

音频编码器通常采用采样、量化、预测、变换和熵编码等技术，将模拟音频信号转换为数字信号，并进行压缩编码。具体来说，音频编码过程主要包括以下几个步骤：

  • 采样：将模拟音频信号转换为数字信号，以便于后续的处理和传输。
  • 量化：对采样后的音频数据进行量化处理，将连续的信号离散化，减少数据冗余。
  • 预测编码：通过对音频信号的历史数据进行分析和预测，来减少音频数据的冗余，包括自适应差分编码、线性预测编码等。
  • 变换编码：通过将时域中的音频数据转换到频域中，并采用离散余弦变换（DCT）或离散小波变换（DWT）进行变换编码，以进一步压缩音频数据。
  • 熵编码：采用霍夫曼编码、算术编码等熵编码技术，将压缩后的音频数据进行进一步的压缩，以减小数据传输带宽。

编码格式

编码格式 是否有损压缩 应用场景 简介 优点 缺点
MP3 有损压缩 音乐、广播等领域 MPEG Audio Layer III，是一种有损音频压缩格式。 文件小，易于传输和存储；广泛应用。 压缩后音质丢失，损失细节。
AAC 有损压缩 数字电视、数字广播、移动通信等领域 Advanced Audio Coding，是一种有损音频压缩格式。 音质较好，压缩率高；被广泛应用。 编码速度慢，兼容性差。
WMA 有损压缩 Windows操作系统、Windows Media Player等产品 Windows Media Audio，是微软公司开发的一种有损音频压缩格式。 压缩率高，音质较好；与Windows系统兼容性好。 在非Windows系统上使用受限。
FLAC 无损压缩 音乐存储和传输领域 Free Lossless Audio Codec，是一种无损音频压缩格式。 压缩后音质不丢失，损失细节；压缩率高。 文件较大，不易传输和存储。
OGG 无损压缩 音乐、游戏、视频等领域 Ogg Vorbis，是一种自由、开放的无损音频编码格式。 压缩率高，文件较小；无版权限制。 兼容性差，压缩率可能不高。
WAV 无损压缩 音乐制作和录音等领域 Waveform Audio File Format，是一种无压缩的音频文件格式。 无损压缩，音质好；广泛应用于音乐制作和录音等领域。 文件较大，不易传输和存储。
APE 无损压缩 音乐存储和传输领域 Monkey’s Audio，是一种高效的无损音频压缩格式。 压缩率高，音质好；支持嵌入式cue信息。 不常见，兼容性差。
ALAC 无损压缩 Apple产品中 Apple Lossless Audio Codec，是一种无损音频压缩格式。 音质好，压缩率高；在Apple产品中得到广泛应用。 不兼容非Apple系统，文件

编码

  • 采样：对原始音频信号进行采样处理，将连续的模拟音频信号转换为离散的数字音频信号。
  • 预处理：对采样后的音频信号进行预处理，包括滤波、降噪、增益调节等操作，以去除噪声和干扰，提高信噪比。
  • 分析：对预处理后的音频信号进行分析，提取其特征，包括声音的频率、振幅、时域和频域特征等。
  • 压缩编码：将分析后的音频信号进行压缩编码，以减小数据量，提高数据传输效率。常用的音频编码格式包括MP3、AAC、FLAC、OGG等。
  • 解码：将编码后的音频数据进行解码还原，得到原始音频信号。
  • 合成：对解码后的音频数据进行合成，包括滤波、平滑等操作，以提高音频质量和音乐感。
  • 输出：将合成后的音频信号输出到相应的设备或存储介质上。

解码

  • 解封装：对音频进行解封装，得到音频流数据。
  • 解码：将音频流数据进行解码，得到PCM（Pulse Code Modulation）数据，即数字音频数据。
  • 重采样：根据音频流的采样率和PCM的采样率的不同，进行重采样处理。重采样是指改变数字音频的采样率，使其与设备的采样率匹配。
  • 播放：将PCM数据传递给音频设备进行播放。
