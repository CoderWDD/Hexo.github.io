介绍

  • HashMap 是 Java 中提供的一种 Map 集合，继承自 AbstractMap，同时实现了 Map 接口
  • 其中 AbstractMap 提供了一套 Map 的基础模板，其中定义了包括 get()、put() 、containsKey() 之类的方法（这也体现了 抽象类是模板，接口是规范 的说法）

问题解答

HashMap 用了什么数据结构？

当单条哈希冲突链表的长度大于 TREEIFY_THRESHOLD 时，就是 数组+红黑树，否则就是 数组+链表

HashMap 线程安全吗？

不安全的，以为在 put 之类的操作时，没有进行相应的同步机制

如果 HashMap 存储的数量超过了设置的上限会怎么样？

  • 如果还没到 Integer.MAX_VALUE，就进行扩容，一般是扩大一倍
  • 如果达到 Integer.MAX_VALUE，就没法扩容

HashMap 中的负载因子是干嘛的？

作为是否扩容的一个决定因子，当哈希表中元素数量达到了负载因子乘以数组容量时，就需要进行扩容操作

HashMap 在使用数组+链表的结构时，新插入的节点是头插入还是尾插入？

尾插

在什么情况下，会从 链表 转成 红黑树 存储？

在单条哈希冲突链表的长度超过 TREEIFY_THRESHOLD 时

在什么情况下，会从 红黑树 转成 链表 存储？

在单个哈希冲突红黑树中的元素个数小于 UNTREEIFY_THRESHOLD 时
