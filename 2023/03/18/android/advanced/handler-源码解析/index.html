Handler_源码解析

Created2023-03-18|Updated2023-03-18|AndroidAdvanced
|Post View:

带着问题学习

  1. Handler 的作用是什么？
  2. 为什么在主线程中可以直接使用 Handler，而不需要创建 Looper？
  3. 为什么在子线程中不能直接使用 Handler，而要先用 Looper.prepare()？
  4. Handler 的 post() 和 send 有什么区别？
  5. Handler 的 postDelay() 是准确的延时吗？
  6. Handler 中的 messageQueue 是什么样的结构？
  7. Handler 中的死循环为什么不会一直占用 CPU 资源？
  8. Handler、Looper、MessageQueue 之间是怎么工作的？

简介

  • Android 开发的世界里，流传着一个说法：只能在 UI 线程中更新 UI
  • 但是我们耗时操作都是在子线程中进行的，而且往往执行完耗时操作，要把耗时操作的结果更新到界面中显示出来，这就避免不了要进行更新 UI 的操作，可是，我们是在子线程中获取到的结果，要怎么才能无痛地将结果显示到界面中呢？
  • 这时候，就需要用到一种 线程间的消息机制：Handler，也就是说，Handler 主要是进行线程间通信的，也就是把 子线程 的消息，发送给 主线程，从而达到 更新 UI 的目的

作用

看看官方的说法

to schedule messages and runnables to be executed at some point in the future
to enqueue an action to be performed on a different thread than your own.

  • 执行定时任务
  • 把任务交给其他线程执行

组成

先来看看 Handler 工作时，其必要的组成是什么。后面一一进行解析

  • Handler
  • MessageQueue
  • Looper

大概原理

Handler 用于将消息或者 Runnable 对象投递到消息队列（MessageQueue）中，然后由 Looper 从消息队列中取出消息并进行处理

使用步骤

具体在使用过程，已经写过相关的文章了，不赘述，下面是相关的步骤

  • 创建 Handler 对象
  • 在子线程中调用 handler
  • 启动子线程
