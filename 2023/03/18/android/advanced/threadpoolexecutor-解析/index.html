ThreadPoolExecutor_解析

Created2023-03-18|Updated2023-03-18|AndroidAdvanced
|Post View:

带着问题学习

  1. 线程池是什么？
  2. 线程池有几种？
  3. 为什么要用线程池？
  4. 线程池的组成是什么样的？
  5. 不同线程池之间有什么差别？

简介

苦于实在没有艺术天分，下面用 ChatGPT 生成的一个故事来引入线程池的各个核心概念

假设我们要在一个工厂中完成一些任务，这个工厂有很多工人和一些设备。为了让任务能够高效地完成，我们需要对这些工人和设备进行合理的管理和分配。

这些工人就是线程池中的 工作线程，他们可以执行各种任务，比如加工、装配、包装等。这些设备就是工作线程需要用到的资源，比如机器、工具等。

任务队列 就像是工厂中的订单，当一个任务到达时，会被添加到任务队列中等待工人来处理。

线程池管理器 就像是一个工厂管理员，负责管理工人和设备，调度工人完成任务，以达到最佳的生产效率。

线程工厂 就像是工厂的人力资源部门，用于创建线程池中的工作线程。可以自定义线程工厂来创建特定类型的工作线程，比如不同等级的工人等。

作用

从上面的例子，可以看出，线程池主要用于 管理和复用 线程，以提高应用程序的性能和响应速度，其优点大致可以总结为以下几点：

  • 提高性能：线程池能够复用已经创建的线程，减少线程创建和销毁的开销，从而提高应用程序的性能和响应速度。

  • 简化编程：线程池可以屏蔽底层线程的复杂性，使得开发人员可以更方便地编写多线程程序。

  • 控制并发数：线程池可以限制同时执行的线程数量，避免线程数量过多导致系统负荷过重。

  • 提高稳定性：线程池可以避免因为线程数量过多导致系统崩溃的情况发生。

核心组成

任务队列

  • 用于存储等待执行的任务，一个先进先出的队列。

  • 当一个任务到达时，线程池会将任务添加到任务队列中，等待线程来执行。

工作线程

线程池中的实际执行者，用于执行任务。

线程池管理器

用于管理线程池中的工作线程，包括线程的 创建、销毁、调度和任务分配 等操作。并根据实际需求动态调整线程池大小，以达到最佳的性能和效果。

线程工厂

用于创建线程池中的工作线程

线程池分类

FixedThreadPool

  • 固定大小 的线程池（corePoolSize = maximumPoolSize），线程数量不会改变
  • 即使线程池的线程处于空闲状态，这些核心线程也不会被回收
  • 如果当前没有空闲线程，即使有新的任务到来，也不会新建线程，而是会让任务处于等待状态
  • 适用于执行长期的任务，性能稳定，不会创建过多的线程

CachedThreadPool

  • 缓存线程池，线程数量根据任务数 自动调整，最大线程数量为 Integer.MAX_VALUE
  • 因为线程数量无限大，所以 任何任务都能被立即执行
  • 核心线程数为 0
  • 因为核心线程数为 0，所以当线程池限制时，线程会被回收，不会占用系统资源
  • 适用于执行大量短期的异步任务

SingleThreadPool

  • 单线程池，只有一个工作线程 的线程池，还没被执行的任务，处于等待状态

  • 适用于需要按顺序执行任务的场景

ScheduledThreadPool

  • 周期线程池，核心线程数固定，非核心线程则没有限制数量
  • 用于执行定时任务和周期性任务，支持延迟执行任务

线程池相关参数

参数名 参数类型 描述
corePoolSize int 核心线程池大小，即线程池中始终保持的线程数量。
maximumPoolSize int 最大线程池大小，即线程池中允许创建的最大线程数量。
keepAliveTime long 线程空闲时间，即当线程池中线程数量超过corePoolSize时，多余的空闲线程在被终止之前等待新任务的最长时间。
TimeUnit TimeUnit枚举类型 时间单位，用于设置keepAliveTime的时间单位。
workQueue BlockingQueue<Runnable>类型 任务队列，用于存储等待执行的任务。
threadFactory ThreadFactory接口类型 线程工厂，用于创建新线程。
handler RejectedExecutionHandler接口类型 任务拒绝处理器，当任务无法被线程池执行时的处理方式，例如抛出异常或者丢弃任务。
