屏幕适配是什么

屏幕适配是指为 不同大小、不同分辨率 的 Android 设备提供优化的布局和资源，以确保应用程序在各种设备上的外观和性能都能得到良好的体验。

为什么需要屏幕适配

市面上流行的 Android 设备五花八门，不同厂商不同型号不同规格的手机，可能在屏幕参数上，都会有差异，如果我们对所有的设备都使用同一套布局，那就可能导致在某些设备上，界面效果不能如我们预期设想的那样了。因此，为了让我们的应用在不同尺寸的设备上，都能有比较统一的表现，需要根据屏幕大小，进行一些适配。

相关概念

PX——Pixel

  • 即像素，常说的手机分辨率：1080 x 1920 指的是手机宽度可展示 1080 像素，高度可展示 1920 像素。

PPI——Pixels Per Inch

  • 每英寸包含的物理像素数，取决于屏幕本身，PPI 值越大，图像显示越清晰。

  • 公式

$$
PPI = \frac{\sqrt{\text{水平方向像素点数的平方 + 垂直方向像素点数的平方}}}{\text{屏幕尺寸}}
$$

DPI——Dots per inch

  • 每英寸点数，用于描述设备的 屏幕像素密度

  • DPI 值越高，意味着图像的分辨率越高，可以更准确地重现图像的细节。

SP——Scalable pixels

会按用户首选的文本尺寸进行缩放（属于缩放无关像素），当用户修改手机显示的字体时，字体大小会随之改变。

适配方案

摆烂式——dp

  • dp：即 density-independent pixel，即密度无关像素，也叫 dip，Android 系统会自动将 dp 值根据当前设备的屏幕密度进行缩放

$$
dp= \frac{像素值}{density} = \frac{px}{(dpi/160)}
$$

  • 缺陷：不能解决所有问题，因为屏幕尺寸多种多样，没法通过 dp 统一解决

比例大法——双边比例

  • 计算出我们的控件在水平方向上和竖直方向上，分别占用屏幕的百分之多少，然后以这个比例来显示我们的控件
  • 这样的做法虽然可以使我们的控件在屏幕上显示的比例不变，但是很可能出现控件本身比例失调的问题
  • 比如，我们要显示一个 200*200 的正方形，当我们在一个 400*800 的屏幕上计算时，我们得到的宽是占屏幕的 50%，高是占屏幕的 25%
  • 当如果我们此时换到了 400*1000 的屏幕上，并仍然用 50% 和 25% 作为宽高比例计算，我们会得到一个 200*250 的长方形，并不是我们一开始预期的正方形

比例大法——单边比例

  • 既然我们在宽高两个方向上按比例进行适配行不通，那试试单边呢？
  • 其实这时控件本身的比例失衡更明显，因为我们如果按宽的比例来计算的话，那意味着我们在控件的高要写成固定的数字，遇到宽点的屏幕，还是会出现把正方形缩放成长方形的情况

今日头条方案

  • dp 与 px 的转换公式：

    • px = density * dp
    • density = dpi / 160
    • px = dp * (dpi / 160)

  • 需要预先明确的点：在布局文件中声明的 dp 值，最终都需要通过 TypedValue 的 applyDimension 方法来转换为 px，转换公式即：density * dp

    而 dp 的值就是我们希望的大小，所以最终显示的大小，跟 density 密切相关，如果我们能够根据不同的设备，来把 density 修改成不同的值，那就可以实现我们的 dp 在不同的设备上，都有相似的表现了，而这个方法完全可以实现，因为 density 本身是 DisplayMetrics 的一个 public 属性，这意味着我们可以对其直接进行修改，而不需要使用类似反射之类的黑科技
