引入

经过了对 MVC 学习和简单应用，我们可以知道，在 Android 中 Activity 就充当了 Controller 的角色，当业务复杂的时候，Activity 会很快膨胀，最终难以维护

MVP 简介

MVP 三个字母分别代表什么

  • M：Model，整个 MVP 的核心，负责数据获取及逻辑处理，与 MVC 中不同的是，此时的 Model 的数据，是给 Presenter 进行调用，而不是给 View
  • V：View，用于处理用户输入和显示，负责用户交互
  • P：Presenter，连接 Model 和 View 层，接收来自 View 的事件，并分发给不同的 Model，最后调用 View 层的方法，更新 View，相当于一个 中间人 的角色

MVP 的作用

  • 在 MVC 的基础上，使各个组成部分之间更加独立

优缺点

优点

  • 从上面的代码中可以看出来，View 层与 Model 层，彻底没有交集了，完全解耦
  • 所有的交互都是通过 接口 来进行，对于测试和维护友好

缺点

  • 因为所有交互都是通过接口进行的，所以会导致 接口数量暴增
  • 随着接口数量暴增，这也意味着代码量会暴增，增加了开发成本
  • 在使用中，需要把 Activity 传递给 Presenter，很有可能会阻碍 Activity 的回收，造成内存泄漏
