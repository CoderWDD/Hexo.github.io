MVC 简介

如果按照上面所介绍的，各种版本都是 MVC，且其核心都是符合 MVC 的设计规范的，那么到底 MVC 的核心思想应该是怎么样的呢？

MVC 三个字母分别代表什么

  • M：Model，整个 MVC 的核心，负责数据获取及逻辑处理，并把结果 直接或间接 返回给 View 进行更新
  • V：View，用于显示数据和接收用户输入，并把事件传递给 Controller，通常理解为应用的界面，在 Android 在通常指 xml 文件
  • C：Controller，作为控制器，处理 View 和 Model 之间的关系

MVC 的设计理论

MVC 的初衷就是把用户事件从 View 流给 Controller，而 Controller 根据 View 流过来的事件，将其分发给不同的 Model，而 Model 负责获取数据，并把数据返回给 View 进行显示

注意点/缺点

Activity 任务过重：从上面的例子中可以看出，虽然 MVC 是分为了三层，但是，在 Android 中具体使用的时候，Activity 承担了 Controller 的角色，而我们进行视图更新的操作，也是在 Activity 中的，也就是说，虽然 MVC 是三层，但是，我们在 Android 中，并不能真的将其当成三个独立的部分单独抽离出来实现，所以，MVC 模式会使得 Activity 同时负责了 View 更新和 Controller 分发事件的责任，如果对于复杂的情况，Activity 的代码会极速膨胀

总结

MVC 模式的设计是合理的，但是在应用到 Android 实际场景中时，就有点力不从心、水土不服了，因此，具体什么时候使用 MVC 模式，还是要看具体的业务场景和需求
